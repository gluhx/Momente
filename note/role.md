# Ролевая модель приложения

## Роли в БД

- admin : управляющий пользователь. Имеет полный доступ к редактированию и созданию таблиц в БД;
- guest : гостевой пользователь. Имеет права на просмотр таблиц в БД;
- man   : основной пользователь. Имеет права на запись, изменение и удаление данных в таблицах БД, напрямую контактирующих с пользовательским интерфейсом.

## Роли в приложении

- admin : управляющий пользователь. На сайте ему отображаются просто таблицы, которые он может напрямую изменять;
- guest : гостевой пользователь. На сайте он может только просматривать фотографии;
- man   : основной пользователь. Может как просматривать, так и редактировать фльбомы с фотографиями.

## Условия доступа

Для _guest_ необходимо будет ввести обычный пароль, который будет привязан к каждому альбому.

Для _man_ необходимо зарегистрироваться и стать пользователем. Если вы будете добавлены в альбом, то сможете его просматривать.

Для _admin_ нужно будет ввести пароль свой пароль пользователя и пароль админа.
